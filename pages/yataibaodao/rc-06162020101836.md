 <p>
  印度与中国军方近来在克什米尔边境对峙，6月15日演变成棍棒与石头齐飞的互相攻击局面，导致双方都出现严重伤亡。这两个都拥有核武器的人口大国，会重演1962年的武装军事冲突吗？
  <br/>
  <br/>
  不是驳火枪战，中印两国军人在长期以来就有边境争议的克什米尔地区互相扔石头、棍棒殴打，当地时间15日闹出人命。据英国广播·公司引述印度官员的话说，印度死亡官兵已增加到二十人。另据印度亚洲国际新闻社报道，解放军有四十三名官兵伤亡，但中国官方没有证实或否认。这是四十五年来中印边境爆发的最严重的军事冲突。
  <br/>
  <br/>
  中印双方都发表书面声明，互相指责对方才是“背信挑衅”，却都强调要透过“对话”解决分歧。
 </p>

 <p>
  新华社报道，中国西部战区新闻发言人张水利声明，6月15日晚，是印军“违背承诺”，再次越线非法活动，“蓄意发动挑衅攻击，引发双方激烈肢体冲突，造成人员伤亡”。
  <br/>
  <br/>
  中方声明坚称，加勒万河谷地区主权历来属于中国，要求印度约束前线部队，不要出尔反尔。
 </p>
 <p>
  英国广播公司（BBC）则报道，印度政府发言人斯里瓦斯塔瓦（Anurag Srivastava）在记者会上表示，印中双方6月6日经过“建设性和有成效”的会晤，达成缓和局势和边界对峙降温的共识，没想到中方做法“偏离”共识。
  <br/>
  <br/>
  斯里瓦斯塔瓦说，6月15日晚间，是中方士兵试图片面改变原有实际控制线，导致暴力冲突。
  <br/>
  <br/>
  尽管双方都把责任推给对方，但双方都说到共同关键字就是“对话”。
 </p>
 <p>
 </p>
 <p>
  <br/>
  <b>
   美专家：大规模武装冲突可能性低
  </b>
  <br/>
  <br/>
  美国前总统小布什时代在国务院担任国务卿办公室南亚事务政策规划顾问的马基（Daniel Markey），刚刚出版了新书
  <br/>
  《中国的西边地平线》（China's Western Horizon: Beijing and the New Geopolitics of Eurasia），探讨北京与欧亚地缘政治发展。他接受自由亚洲电台访问时说，印度和中国这一次的边境冲突有吵有闹，双方甚至闹出了人命，但两国现在看来都极力克制，不想让局势失控。
  <br/>
  <br/>
  “我不认为会因为这次事件，导致中印双方真的开战......虽然这确实是四十五年来首见的紧张局势，但这和印度与巴基斯坦的火炮空袭，长期以来的武装攻击是不一样的。”马基说。
  <br/>
  <br/>
  印度和巴基斯坦的边境与领土争议，从未消停。去年二月，双方出动军机空袭交战；另一方面，巴基斯坦长期以来和中国交好，这让印度在边境问题上腹背受敌。
 </p>

 <p>
  <b>
   美中亚洲争锋 印度“新不结盟运动”持盈保泰？
  </b>
  <br/>
  <br/>
  美国智库东西中心（East West Center） 华盛顿办公室副总裁利马耶（Satu Limaye）和马基有相似观点。
  <br/>
  <br/>
  利马耶告诉本台，中印两国如果真会重演1962年的边境武装冲突，会让他很意外，他更关注的是美中两强在印太区域的交锋。印度为自身国家利益考虑，新德里有的变与不变，也反映了东南亚国家看待美中冲突的自身视野。
  <br/>
  <br/>
  利马耶说，“印度正逐步改变过去一向和美国及美国的盟邦保持等距交往的姿态。但我会说，这是’新不结盟运动’、不是抛弃过去的’不结盟运动’。”
  <br/>
  <br/>
  马基和利马耶都指出，南亚地区的和平稳定是美国的利益，然而，利马耶表示，中国和印度在涉及主权议题上歧见如此之深，但是双方却都拒绝了美国总统特朗普提出的愿意出面调停的提议。
  <br/>
  <br/>
  利马耶表示：“必须厘清的是，就算印度在外交上、政治上与军事上和美国及美国的盟邦走得更近了，也不代表印度会切段所有和中国的经济往来，或和中国发生冲突。没有哪一个印太国家想和中国发生军事冲突。”
 </p>
 <p>
  马基目前是约翰·霍普金斯大学高等国际研究院资深教授，他则认为，美国展现一定姿态还是必要的，相信多个正承受着中国崛起压力的亚洲国家，也都看在眼里，“不能让中国认为，反正美中关系已经'破罐子破摔'、坏到不能再坏了，在印度与香港的争议上又生事端，他们不需要付出额外代价。美国必须清楚表明，中国再有这类侵略性的行动，是要付出代价的。”
  <br/>
  <br/>
  他还说，美国可以透过多种方式协助印度，例如提供卫星图像资讯与情报，让印度更清楚究竟中国在当地做什么。
 </p>
 <p>
  截至发稿，美国国务院没有回复自由亚洲电台置评的请求。
 </p>
